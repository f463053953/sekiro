# Vue 中一些必须遵循的规则

# 优先级 A：必要的

1. 组件名称为多个单词
为避免组件名称和 HTML 元素冲突，所以将组件名称设置为多个单词
2. 组件的 data 必须为一个函数
当在组件中使用 data property 的时候 (除了 new Vue 外的任何地方)，它的值必须是返回一个对象的函数。

Vue.component('some-comp', {
  data: function () {
    return {
      foo: 'bar'
    }
  }
})

3.Prop 定义应该尽量详细
它们写明了组件的 API，所以很容易看懂组件的用法
在开发环境下，如果向一个组件提供格式不正确的 prop，Vue 将会告警，以帮助你捕获潜在的错误来源

props: {
  status: String,
  data:json,
}

4. 为 v-for 设置键值

<ul>
  <li
    v-for="todo in todos"
    :key="todo.id"
  >
    {{ todo.text }}
  </li>
</ul>

5.v-if 和 v-for 不要同时使用在一个标签上
6. 为组件样式设置作用域
主要针对大项目开发时，或引入第三方 HTML/CSS 设置作用于保证样式不会冲突
